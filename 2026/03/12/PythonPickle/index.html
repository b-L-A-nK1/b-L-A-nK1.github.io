Pickle反序列化

Pickle本质上是一种编程语言，在调用pickle.loads()时其实是在启动一个名为PVM的虚拟机

这个虚拟机由三部分组成

但是由于PVM的指令集中包含了执行函数的指令。所以就可以对执行流进行伪造，PVM就会在栈上组装出恶意函数和恶意参数，并执行。

1、脚本生成

当题目对传输内容几乎没有对格式的过滤时就可以使用脚本自动生成

这里的__reduce__是Python中的一个魔术方法，在类中它会要求至少返回一个函数和参数

当对这个类进行Pickle序列化的时候，传输过去又进行反序列化就会告诉服务器执行这个附有参数的函数

但是有的时候题目会过滤一些不可见字符，而这些不可见字符在进行序列化的时候肯定会出现的

这是因为pickle.dumps()默认用的是Protocol4/5版本，这两个版本具有打包数据体积小、速度快的特点，也就是使用了许多不可见字符辅助的原因。

所以如果题目中出现了对这些特殊字符的过滤，可以将protocol参数设置为0

2、手动注入

首先要了解常用的Opcode字典

指令 含义 栈操作描述 格式要求
c GLOBAL 引入模块和属性。压入栈顶 c<模块名>\n<属性名>\n
( MARK 在栈顶放一个标记（像一个挡板） (
V UNICODE 引入一个Unicode字符串。压入栈顶 V<字符串>\n
S STRING 引入一个字符串(带引号)。压入栈顶 S’<字符串>’\n
t TUPLE 从栈顶向下寻找(标记，把中间的所有元素打包成一个元组。压入栈顶 t
R REDUCE 弹出栈顶的元组作为参数，弹出其下面的函数并执行。将执行的结果压入栈顶 R
. STOP 结束反序列化，返回栈顶元素 .
o OBJ 创建空对象 o
} EMPTY_DICT 压入空字典 }
s SETITEM 给字典赋值 s
a APPEND 给列表追加元素 a

但是在手动注入这里要十分小心，有的参数因为是单字节操作后面不可以跟着换行符(\n)，不然PVM那边会出现解析错误,所以手动注入这边推荐写成一行

这里可以大致分为

1、需要加两个或多个\n
c i
2、需要加一个\n
V S I p/g
3、不可以加\n

这类是操作指令，在PVM字典中只占1字节

( t R b . o } s a

严格过滤情况

当题目如果把b’R’过滤的话，可能认为就没有办法去执行指令了，但是PVM中还有两个可以用用来执行的参数

1、i(INST)指令

语法：i<模块名>\n<函数名>\n

工作原理：它会像t一样往下寻找MARK挡板，把挡板上面的所有东西当成参数，然后直接去调用所指定的函数

相当于执行subprocess.check_output(‘ls /‘)

2、b(BUILD)状态劫持

b(BUILD)的设计初衷是恢复对象的状态（属性）

栈结构要求：[某个对象实例，状态数据]

当PVM遇到b是，会做这两步判断：

1、检查：这个对象有没有__setstate__这个魔术方法

2、执行：如果有,PVM就会强制执行 对象._setstate_(数据状态)

3、备用机制：如果没有这个方法，PVM就会默认去更新对象的字典属性: 对象._dict_.update(状态数据)

所以这里就可以对一个对象，利用b的备用机制，在它的字典里面塞入一个”_setstate_”: subprocess.check_output

然后再对这个对象用一次b，PVM就会执行指令了

但是由于b参数并不会关心返回值而是直接仍了，所以是看不到执行结果的返回值的

页面注入后只会看到这个

所以要用到反弹shell了
